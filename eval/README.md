# eval

Evaluation harness for the SPEC §8.1 metric suite. (The directory name is project convention — the spec mandates the artifacts, not this exact path.)

평가 하니스. 평가 전용 500세션(홀드아웃 지도 2종 + 경화 슬라이스 포함, 이력·학습과 시드·지도 분리 §2.7)에서 전 지표를 산출한다.

## Contents

- `quick100.py` **(live, M0)** — 100 eval sessions (seed 4242, disjoint from history): time MAE, T_req MAE, consumption MAPE, 90 % interval coverage. Charge-resume sessions are excluded from clean-discharge scoring (§3.1). `--check` is the CI regression gate (time MAE ≤ baseline ×1.10, §12.2); `baseline_quick100.json` is committed.
- `eval500.py` **(live, M2)** — the full §8.1 suite over 500 sessions: 5 SoH cohorts × 100, each fitting a matched-aging history (train maps only) and evaluating fresh sessions of the same SoH across all 5 maps. Reports **time MAE (decision band)** ≤ 2.0 · **T_req MAE** ≤ 2.0 · **MAPE** ≤ 12 % · **coverage** 86–94 % · **state agreement** ≥ 90 % · **aging (SoH 0.8) MAE** ≤ 2.5, plus hardened slices (AR(1) / holdout maps / mode-change / charge-resume, the last two as segment-rate MAPE per §3.1). Time MAE is banded by start battery because it scales with the battery range projected — the low-battery decision band is the headline, MAPE is the scale-free one. `--quick` is the pytest smoke run.
- Committed artifacts (regenerated by `make eval500`, byte-stable — fixed seed): `report_eval500.json`, `eval_report.md`, **`reliability_diagram.svg`** (10-bin predicted P vs observed frequency — "our 92 % is really 92 %"), **`ablation_table.md`** (raw → +EWMA → +shrink → +drift; RLS/LSTM rows land in M4).
